01 / Overview

Three learning paradigms, one agent

The HeroBot task is a sandbox for the three classical branches of machine learning. Rather than treat them separately, the coursework runs all three on the same agent: it has to perceive its world (supervised), organise what it finds (unsupervised), and act to reach a goal (reinforcement). Each part is framed as a controlled comparison between two methods so the trade-offs are explicit rather than assumed.

  • 1Supervised. Classify 80×80 sprite tiles into their entity class — a CNN versus a KNN built on HOG features.
  • 2Unsupervised. Cluster entities from a feature table — K-Means versus agglomerative hierarchical clustering.
  • 3Reinforcement. Learn a navigation policy on a 10×10 dungeon maze — Q-Learning versus SARSA.

Reproducibility: RANDOM_SEED = 42 throughout; the raw sensor log (dungeon_sensorstats.csv) is treated as read-only and never overwritten.

02 / Supervised

Seeing the world: sprite classification

The agent has to label 80×80-pixel sprite tiles by entity type. I implemented two contrasting approaches:

Algorithm A — Convolutional Neural Network

A three-block convolutional network with batch normalisation and global average pooling, learning its own features end-to-end from the raw pixels — the modern, representation-learning route.

Algorithm B — KNN on HOG features

A k-nearest-neighbours classifier over Histogram-of-Oriented-Gradients descriptors — a lightweight, interpretable, hand-engineered-feature route that needs no training loop.

The comparison is the point: the CNN trades interpretability and compute for the ability to learn features, while KNN+HOG trades capacity for transparency and speed. To make the comparison rigorous I went beyond raw accuracy — using McNemar's test to check whether the two classifiers' error patterns differ significantly, and expected calibration error (ECE) to ask whether the CNN's confidence scores are trustworthy, not just its top-1 labels.

03 / Unsupervised

Organising the world: entity clustering

Given a feature table describing the entities the agent encounters, the task is to recover structure with no labels. I compared K-Means (centroid-based, fast, assumes roughly spherical clusters) against agglomerative hierarchical clustering (linkage-based, makes no cluster-shape assumption and yields a full dendrogram).

Choosing the number of clusters is the hard part of unsupervised work, so I used the gap statistic to select k in a principled way rather than by eye, and assessed cluster stability across resamples to check that the discovered groups are real structure and not an artefact of one particular split.

04 / Reinforcement

Acting in the world: maze navigation

The final part puts the agent in a 10×10 dungeon maze and asks it to learn a path to the goal from reward alone. I implemented two tabular temporal-difference learners:

MethodUpdate styleCharacter
Q-LearningOff-policyLearns the greedy optimum; more aggressive near hazards
SARSAOn-policyLearns the policy it actually follows; more cautious

Because reward in a sparse maze only arrives at the goal, I used potential-based reward shaping (Ng et al., 1999) to densify the signal without changing the optimal policy — the factorial structure of the shaping potential lets the agent learn faster while preserving the guarantee that it is still solving the original problem. The on-policy / off-policy contrast then shows up exactly where theory predicts: SARSA's learned route keeps a safer margin from penalty tiles, while Q-Learning's takes the shorter, riskier line.

05 / Takeaways

What I took from it

  • "Which algorithm is better" is the wrong question — each pairing (CNN/KNN, K-Means/hierarchical, Q-Learning/SARSA) is a trade-off, and the job is to name the axis it trades on.
  • Top-1 accuracy alone is not enough: McNemar, ECE, the gap statistic, and stability analysis are what turn a number into a defensible claim.
  • Reward shaping is a safety-relevant design choice — using a potential-based form is what guarantees the densified reward still optimises the intended behaviour.
